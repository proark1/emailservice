# Self-Hosted VPS Deployment Guide

Deploy MailNowAPI on your own server with full email send/receive capability.

## Requirements

- **VPS**: Hetzner CX22 (2 vCPU, 4GB RAM, €4.35/mo) or any VPS with port 25 open
- **OS**: Ubuntu 24.04 LTS
- **Domain**: A domain you control (e.g., `yourdomain.com`)

## Architecture

```
Internet
  │
  ├── :443 → Caddy (auto HTTPS) → :3000 (API + Dashboard)
  ├── :25  → Postfix (send outbound / receive inbound)
  ├── :587 → App SMTP relay (clients send with API key auth)
  ├── :465 → App SMTP relay (implicit TLS)
  └── :80  → Caddy (ACME challenges + redirect)

Internal:
  App → localhost:25 (Postfix) → recipient MX servers
  Postfix → localhost:2525 (App SMTP inbound) → stored in DB
```

## Step 1: Create Server

1. Sign up at [hetzner.com/cloud](https://www.hetzner.com/cloud)
2. Create a **CX22** server with **Ubuntu 24.04**
3. Note the server's **IPv4 address** (e.g., `1.2.3.4`)
4. **Request port 25 unblock**: Open a Hetzner support ticket asking to remove the SMTP block — they approve in minutes

## Step 2: Configure DNS

At your domain registrar, add these records (replace `1.2.3.4` with your server IP):

| Type | Name | Value |
|------|------|-------|
| A | `yourdomain.com` | `1.2.3.4` |
| A | `mail.yourdomain.com` | `1.2.3.4` |
| MX | `yourdomain.com` | `10 mail.yourdomain.com` |
| TXT | `yourdomain.com` | `v=spf1 a mx ip4:1.2.3.4 -all` |
| TXT | `_dmarc.yourdomain.com` | `v=DMARC1; p=none; rua=mailto:dmarc@yourdomain.com` |

**PTR record** (reverse DNS): Set in the Hetzner Cloud panel → Server → Networking → set rDNS to `mail.yourdomain.com`

**DKIM**: Will be generated by the app when you add your domain. Add the TXT record it provides.

## Step 3: Deploy

SSH into your server and run:

```bash
# Clone the repo
git clone https://github.com/YOUR_REPO/emailservice.git
cd emailservice

# Run setup (installs Docker, Caddy, Postfix, configures everything)
sudo bash deploy/setup.sh yourdomain.com mail.yourdomain.com

# Review the generated .env file
nano .env

# Build and start all services
docker compose -f deploy/docker-compose.prod.yml up -d --build

# Check status
docker compose -f deploy/docker-compose.prod.yml ps
docker compose -f deploy/docker-compose.prod.yml logs -f app
```

## Step 4: Seed & Test

```bash
# Create initial admin account + API key
docker compose -f deploy/docker-compose.prod.yml exec app node dist/scripts/seed.js

# Test the API
curl https://yourdomain.com/health

# Add your domain via API (prints DNS records to configure)
curl -X POST https://yourdomain.com/v1/domains \
  -H "Authorization: Bearer es_YOUR_KEY" \
  -H "Content-Type: application/json" \
  -d '{"name":"yourdomain.com"}'

# Send a test email
curl -X POST https://yourdomain.com/v1/emails \
  -H "Authorization: Bearer es_YOUR_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "from": "hello@yourdomain.com",
    "to": ["your@gmail.com"],
    "subject": "Test from MailNowAPI",
    "html": "<h1>It works!</h1>"
  }'
```

## Step 5: Verify Email Deliverability

1. Send a test email to [mail-tester.com](https://www.mail-tester.com) — aim for 9+/10
2. Check that SPF, DKIM, and DMARC all pass
3. If score is low, verify:
   - PTR record matches `mail.yourdomain.com`
   - SPF TXT record is correct
   - DKIM record is published (check in dashboard → Domains)

## Monitoring

```bash
# App logs
docker compose -f deploy/docker-compose.prod.yml logs -f app worker smtp-relay

# Postfix mail queue
sudo postqueue -p

# Postfix logs
sudo tail -f /var/log/postfix.log

# Caddy logs
sudo tail -f /var/log/caddy/access.log
```

## Troubleshooting

**Emails not sending?**
- Check Postfix queue: `sudo postqueue -p`
- Check Postfix logs: `sudo tail -100 /var/log/postfix.log`
- Verify port 25 is open: `telnet gmail-smtp-in.l.google.com 25` (should connect)

**Emails going to spam?**
- Check PTR record: `dig -x YOUR_IP` should return `mail.yourdomain.com`
- Check SPF: `dig TXT yourdomain.com` should include your IP
- Check DKIM: send to mail-tester.com
- Wait 24-48h for new IP reputation to build

**Can't receive emails?**
- Check MX record: `dig MX yourdomain.com` should point to `mail.yourdomain.com`
- Check Postfix is listening: `sudo ss -tlnp | grep :25`
- Check Postfix can reach the app: `telnet 127.0.0.1 2525`

**502 Bad Gateway?**
- App not running: `docker compose -f deploy/docker-compose.prod.yml ps`
- Check app logs: `docker compose -f deploy/docker-compose.prod.yml logs app`
